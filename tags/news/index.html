设计每周清单：天猫品牌全新升级、Framer发布新版本、Skype全新改版

设计每周清单专注于互联网设计领域内容，分为热点资讯、软件工具、设计经验、案例研究、网站推荐、深度思考等栏目。本文同时发布于知乎专栏「Designer News」，欢迎关注。

热点资讯

  1. 天猫品牌全新升级：在上一周5月23日，天猫 slogan 正式对外更名「理想生活上天猫」，相对之前「上天猫就够了」变得更加聚焦。同时，对品牌进行了全新升级。
  2. Skype全新改版：新功能，新外观，和目前大部分App改版一样，极简外观似乎成为了趋势。
  3. Framer发布新版本：终于在31日，Framer发布了新版本。作为一个原型软件，这一次将设计、原型、代码完美结合，让懂前端代码的设计师更充分发挥设计能力。 / DN讨论
  4. Apple 官方设计资源更新了 Adobe XD 文件：Apple UI Design Resources是苹果今年推出的官方设计资源，包含Sketch、Photoshop文件，目前更新了Adobe XD文件。
  5. Google 新 Emoji 表情 ：Google 在2017年 I/O 中发布的全新 emoji。/知乎讨论

软件工具

  1. Wake：一个分享设计工作与收集反馈的团队协作工具，开始为小团队提供免费版服务。
  2. Feather：一套简洁的开源线性ICON。

设计经验

  1. 各类移动端导航菜单的优缺点：讨论了时下几种移动端导航模式的优缺点：Hamburger Menu 侧栏导航、Tab Bar 底部导航、Priority+ Pattern 优先级动态菜单、Floating Action Button 浮动按钮、Full-Screen Navigation 全屏导航菜单、Gesture-Based Navigation 手势导航、3D Touch、Innovative Navigation Patterns 创新导航模式。
  2. 3种方式从Sketch导入到Framer：Framer发布新版本受到了广泛关注，很多人都表示已经以Sketch作为主力设计工具，那么如何与Framer共同使用呢？官方没有给出合适的方案（大概是要立志取代Sketch作为主力设计工具吧…）。这篇文章介绍了3种目前的方法，从Sketch导入到Framer，以及其优缺点对比。
  3. 麦当劳通用营养标识ICON：麦当劳为109个国家设计的通用营养标识ICON。
  4. 如何用AE制作粒子页面特效：使用Trapcode Particular粒子插件，在AE上制作粒子特效。

网站推荐

  1. Apple Accessibility：苹果的无障碍设计。

深度思考

  1. 设计师 Bill Kenney 专访：设计师出身的Bill，创立了Focus Lab公司，管理团队。

设计每周清单：菜鸟发布新品牌标识、Sketch原型预览工具Mirr.io、Google I/O 2017 Design Talks

设计每周清单专注于互联网设计领域内容，分为热点资讯、软件工具、设计经验、案例研究、网站推荐、深度思考等栏目。本文同时发布于知乎专栏「Designer News」，欢迎关注。

热点资讯

  1. 菜鸟发布了新的品牌标识：5月22号在全球智慧物流大会上，菜鸟发布了新的品牌标识。
  2. Sketch插件 Craft 的 Library 库功能更新：优化界面，使用更加方便。
  3. Framer 5月31号将迎来重大更新 其称为Design Revolution，将改变原型工具这个领域。

软件工具

  1. Mirr.io：支持同WIFI下直接手机打开链接同步预览。相比Craft需要在InVision上预览需要翻墙，可以尝试下这个，简单方便，缺点是操作起来没有Craft直观。

设计经验

  1. App 设计中，慎用左右横滑设计：介绍了一些左右横滑需要注意的点，所谓的“左右横滑”即“Horizontal Scrolling Lists”。用于在同一个页面的 X 轴方向拓展内容空间，“左右横滑”的交互方式被广泛应用于各类 App 中。
  2. Figma 适应布局的使用：介绍了如何使用约束完成自适应布局。Figma或许是一个不错的原型工具选择，它基于Web，可以在Windows上使用。感兴趣的可以阅读这篇安利。
  3. Craft原型功能使用教程：感兴趣的还可以查看这个视频教程，原型功能很不错，不过如果要线上预览的话要上传到InVision，不翻墙体验稍差。
  4. Google I/O 2017 Design Talks：刚刚过去的Google I/O大会，其中的Design Talks部分，包含7个主题的分享。
  5. 表单设计：一页只做一件事：放弃聚合展开式表单，而选择单页独立步骤表单的优势。感兴趣的可以阅读可乐橙的这篇译文。

网站推荐

  1. 1小时学习AI绘制ICON基础：比较基础的教程，介绍如何绘制线性ICON，比较系统，篇幅较短，总共加起来才只有一小时。只需简单注册就可以观看，重点是免费，没有墙。
  2. Google Doodles：查看历史Google Doodle，包含其创作思路，干货满满，适合研究节日创作，及其卡通动效表现。

深度思考

  1. 如果想要创新，那么别被数据束缚：我们周围充满了各种数据，数据不是我们做事情的驱动，而是要通过数据发掘更多的可能。

设计每周清单：Sketch 44 版本更新，Google I/O 2017 AI First战略，交互原型工具清单

设计每周清单专注于互联网设计领域内容，分为热点资讯、软件工具、设计经验、案例研究、网站推荐、深度思考等栏目。本文同时发布于知乎专栏「Designer News」，欢迎关注。

热点资讯

  1. 2017谷歌I/O大会全面转向AI First：在这次Google I/O 2017正式宣布谷歌战略从 Mobile First（移动先行）全面转向 AI First （人工智能先行）。对于互联网设计行业是否会带有一定的影响呢？例如 Google Assistant 中类似Siri一样的对话式 UI 交互，chen。
  2. Sketch 44 版本更新：改进了Artboard的选择切换，并且支持Artboard内容大小自适应调整； 带来了自适应布局功能，可以减少对AutoLayout等自动布局插件的依赖；支持管理丢失的字体，替换为所选字体；改善了圆角处理方式，过窄过宽的角也能完美适应。
  3. Google公开了Auto Draw的所有数据样本 超过1500万用户在 之前Quick, Draw! 中贡献了这5000万绘图数据样本，用于训练 Auto Draw（根据绘制的图形，自动修正提供绘制好的图形的工具）。

软件工具

  1. SVGito：专门针对于Sketch导出的SVG文件，可以压缩文件大小的同时保持设计样式不改变。
  2. 交互原型工具清单：列出目前60+交互原型工具，大多都是可以快速制作原型、动效、导出到开发等。但推荐影响力比较大的几个：Adobe XD、Principle、ProtoPie、Axure、Flinto、Framer、InVision、UXPin、Kite Compositor、墨刀、Origami Studio、Craft、Launchpad、Webflow。
  3. Craft插件：Sketch插件，这周更新版本后支持直接在 Sketch 上制作原型。并且可以同步设计稿到InVision用于线上展示。
  4. Brand.ai：设计组件管理系统，可以轻松从库中拖动组件于Sketch中设计，同时支持导出到开发并自动同步样式。配合 UX Power Tools 使用效果更佳。

设计经验

  1. 移动端横向列表最佳实践：一些关于横向List需要注意的点：前后遮显关系的视觉暗示，使用“查看所有”按钮，响应式设计适应pad端浏览，在pad端只显示有限内容，滑动时会自动吸附到端点。
  2. AI矢量icon绘制技巧：介绍了一些不错的画icon小技巧。例如画爱心，一般人会想到勾外轮廓，实际上只需要三个锚点就可以了。而且这种方法还有个优势，可以直接调成贝塞尔曲线，即可改变爱心的弧度。另外可以查看作者收录的更多绘制技巧gif。
  3. 设计文件（Sketch）命名规范：第二部分细致说明了Sketch文件的命名以及它的图层的整理。
  4. 又一篇VR设计师指南：系统的列出了设计方法、设计工具、交互基础。
  5. Sketch动态ICON：设计一套ICON，和颜色模版，可以动态地修改ICON并使用不同的颜色。

案例研究

  1. Spotify 交互分析和Redesign：对Spotify应用进行分析，然后进行了自发性的Redesign，是个不错的案例研究。

设计灵感

  1. 聊天式UI：illusion.ai——提供对话式UI交互的工具。对比于常规网站的静态的内容呈现，这里采用了对话式气泡UI进行交互，动态展示用户需要的内容，有点类似游戏类h5的选项对话。让冰冷的网站更加亲切拟人化，应该可以适用于某些场景。
  2. 滑动多选交互：很多App都支持轻扫删除操作，是否可以有一个滑动多选的操作呢？链接中展示了在list上侧滑的同时上下滑动，选择更多内容一起执行操作的交互。

网站推荐

  1. Your Majesty：对一些Redesign后的公司品牌作介绍，如：Samsung SmartThings 、adidas 、BMW 等。
  2. UIPixels：免费Photoshop&Sketch源文件下载：Mockup / UI Kit / ICON

深度思考

  1. 肖勇：设计教育不应停留在授“鱼”和“渔”：08奥运会奖牌“金镶玉”的主创设计师，央美设计学院教授，肖勇的访谈。

设计每周清单：微软设计语言Fluent Design、谷歌推出Material组件内容、通过Sketch插件提高效率

设计每周清单专注于互联网设计领域内容，分为热点资讯、软件工具、设计经验、案例研究、网站推荐、深度思考等栏目。本文同时发布于知乎专栏「Designer News」，欢迎关注。

热点资讯

  1. 苹果推出“如何用 iPhone 拍照”系列视频教程：总过16个短视频，是比较面向大众的基本教程，教程从简单的关掉闪光灯拍照，到比较进阶的垂直全景或逆光（HDR），再到 iPhone 7 人像照（背景虚化），以及一些取景或构图教程，可见苹果其用心所在。
  2. 微软推出全新设计语言 Fluent Design： 5月11日，微软在西雅图举行的Build开发者大会宣布了用于跨设备构建用户界面的设计系统 Fluent Design ，其强调5个元素：Light、Depth、Motion、Material、Scale。详情可观看官方宣传视频，以及阅读《Meet Fluent, Microsoft’s New Design System》、《一次看懂Fluent Design》、知乎讨论。
  3. 谷歌推出 Material 组件内容：在自己material.io网站推出了Material Components页面，包含Android、iOS、Web三个平台，遵循Material Design，模块化、自适应的UI组件。详细可阅读 Designmodo 上这篇 文章。
  4. Sketch 发布44 Beta版本：拥有自适应功能，减少了对自适应插件如Auto-Layout的依赖。不过依然还有不足，详情可阅读Medium上的这篇 文章。

软件工具

  1. Confetti：随机打散元素的Sketch插件，制作类似彩带背景。
  2. Launched：可以直接在Sketch上发布网站的插件。无需代码，一键生成并发布网站；可视化操作添加链接、表单、视频；支持自定义域名绑定；支持Google Analytics等网站分析服务。

设计经验

  1. 东京的导航设计：在日本东京，拥有着复杂的交通系统。而走在东京路上随处可见的步行导航设计，其精细程度，一定有让游客十分惊讶。这篇文章分析了日本东京的交通导航设计，感兴趣也可以阅读可乐橙的译文。
  2. 如何通过编写Sketch插件提高设计效率：发现设计过程中存在的某些需求，通过使用或编写插件，可以有效提高设计效率、节省时间。未来Sketch即将迎来全新的文件格式，变得更加开放透明，对插件编写者以及整个生态圈更加友好，一起期待吧。
  3. 通过Sketch插件Launched，一周末制作一个网站：简单快捷，制作这样一个网站。

案例研究

  1. Apple Music Redesign：觉得iOS自带的Apple Music不好用？那就自己Redesign一个吧。

网站推荐

  1. Nick Jones个人网站：采用概念式的滚动/滑动交互，灵感来自“黄金螺旋”（斐波那契螺旋线），流畅的曲线运动让人根本停不下来。除了电脑上鼠标滚动体验外，难能可贵的是，在移动端触屏体验也十分优秀流畅。
  2. 开始你的音乐创作：通过一系列交互课程，学习基础的音乐创作。不需要准备什么，直接在浏览器上完成一切，创作属于你自己的音乐。

深度思考

  1. 存在永恒的产品设计吗？：好的设计经得住时间考验，例如谷歌搜索、保时捷911，设计样式几十年几乎没有变化。

设计要闻-2017.2

17年2月设计要闻，月中一篇Behance 2017设计趋势传得火热，很长一篇，不过讲得全的同时也很泛，私以为趋势这种东西，仁者见仁，智者见智。不过在国内外众多趋势类总结文章中，有些的观点还是很独到的，例如国内大宝的一篇趋势预测。

另外，网站在移动端的响应式处理已经存在很久，但是少有人提到通常网站的这种响应式处理会让其在移动端中显得过长，如何较好地处理这个问题，值得一番研究。

最后，如果了解的话应该会知道，程序员热衷一种叫做 git 的项目版本控制工具，但设计师却很少会用上，因为psd等设计源文件太大，很难进行上传和回档操作。Abstract这个新软件的出现或许会有些期待吧，虽然目前只能是完美支持Sketch。真希望某一天adobe自家出一个PS/AI的版本控制功能。

设计要闻-2017.1

17年1月设计要闻，这个月涌现了很多关于AI（人工智能）如何影响着设计的热点。对比淡下去的VR设计，这可能更值得关注。关于人工智能，当然不可能取代设计师，而如果仅仅作为辅助设计的话，我觉得不应该对其排斥，至少Adobe MAX大会上谈及的很多项目都与此有关。可能你现在就使用着Adobe系列软件中相关的功能而并未发觉呢。

PaintChainer就是一个很有趣的工具，只需上传线稿并标记上色区域，就可以自动上色。这是一位日本的工程师在15年6月公开的深度学习框架Chainer，还有Python借口的OpenCV（Intel的视觉库），通过深度学习网上60万张已上色的插画师作品来学习上色方式。有兴趣可以看下知乎相关的讨论。

目前设计工具的数量也变得多了起来，虽然我们使用最多的还是PS、AI这些，但不妨多了解一下，《2017年设计工具形势》这篇文章中就有分析讲述。

另外，Dribbble 被 Tiny 收购，这几年Dribbble一直不温不火，推出的新功能也只有Playbook，貌似去年说好的Training栏目也一直未推出。对Dribbble团队来说，被收购以后或许可以双赢发展得更好吧。
