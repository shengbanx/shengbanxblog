设计师，处女心

有这么一个段子：

现在的小姑凉啊，在微博朋友圈写点自己和男生那些事，男生也不叫男生，全是“金牛男”、“天蝎男”、“射手男”，整得跟圣斗士勇闯黄金十二宫似的。还说这样方便快速定位形象。怎么能这样！那么我就要问了，学挖掘机到底哪家强……(中毒已深= =）

发现女生都喜欢看星座。嗯，即使在互联网圈子，处女座设计师也是常黑的对象。

加上之前在看到的这篇“处女座设计师的一封信”,让我想起了处女座和设计师的“不解之缘”：似乎很多设计师都是处女座啊。

小时候吃大碗的炒饭的时候，我会用调羹压实那些饭，从一侧挖到底，慢慢挖着吃，比砌墙还平整，觉得很好玩从不觉得有什么奇怪。直到前段时间和朋友出来吃个芝士焗饭，自自然然地这样吃着，并没觉得奇怪。但和朋友聊着设计的事情，聊着聊着发现自己终于做了一回处女座应该做的事了/哭。

对处女座的评价一般都是“追求完美”、“洁癖”，而我从小就是随和，桌柜里永远是很凌乱的，找个试卷找半天的那种。

虽然我的书桌里面试凌乱的，但是书桌上摆着的书，书脊一定要对齐的，有一本突出来我也要弄一下。那时看来似乎大家都这样，爱整洁嘛，嘛，嘛……只是我刚好是处女座而已。

小时候，绑个鞋带绑半天被老妈说了很多次，画画的时候常常是老画不出自己想要的那种橡皮擦，擦擦擦。追求削苹果皮不要断，核桃弄开取出完整的，玩游戏时升级随机加的属性必须要最好的。从前还不觉得自己的这些有什么奇怪，现在处女座被黑得多了，也发现，原来我刚好是个处女座。

设计师有很多处女座吗？好像是的，即使不是，也有一颗“处女心”。

作为一名设计师，渐渐地也发现自己，

  1. 做设计前会算好参考线，各个组件的大小，是做ios应用时尺寸一定要按照苹果给的规范来，不能差分毫。
  2. 对参考线一定要对准，差一像素都不行。
  3. 有个想法要做作品前，一定会开Xmind来做脑图整理想法。
  4. 用到的颜色一定要统一。分组要按组件部位命名分好。
  5. 做icon时会用理科思维计算好几何图形的尺寸。

拥有一颗处女心的设计师，并不是坏事。最起码的，对待设计比一般人严谨，出错更少。

下面说一下我平时钟爱的PS拓展插件（好用到简直不是人！）：

  1. 参考线插件GuideGuide
  2. 统一项目颜色管理Prisma
  3. 一键管理所有图层Layrs

安装插件前，对应Photoshop版本安装Extension Manager哦。如：CC版本要先安装Adobe Extension Manager CC。CS6版本就安装CS6版本的。

最后，那么问题来了:)

《色彩设计的原理》读书笔记

实习的时候深刻地知道自己色彩上理论基础的不足。特地买回来了这本书，日本的伊达千代写的《平面设计原理》。很喜欢，给我很大启发。下面是自己整理的笔记。

色彩的组成

色彩分为“光源色”、“透过色”和“反射色”。

如果光源色的颜色不同，被其照射而看见的“反射色”也随之变化。太阳光照射下的物体颜色最自然，而阴影下物体的颜色会看起来较为偏蓝；白炽灯照射下会显得偏红，因为白炽灯的波长集中在红色和橘色的区间；日光灯的蓝色波长和你抢那个，物体的颜色看起来偏蓝。

光加上颜色之后所透出的色彩为“透过色”；光照射到物体上反射出来的颜色为“反射色”。

  1. 透过色是以“加色混合”的方式，由红、绿、蓝（RGB）混合，表现出各式各样的颜色。
  2. 反射色是以“减色混合”的方式表现，由青（cyan）、品红（magenta）、黄（yellow）、黑（black）（CMYK）互相结合而成。

色彩是一种不确定的东西，我们常常描述色彩会采用“印象”的说法。如：天空蓝，橘黄色。

色彩三要素：色相、纯度（饱和度）、明度。关于色相，最经典的就是色相环（每人必备)。

“三要素”的运用

三招简易做到显得协调的配色：色相一致，纯度一致，明度一致

  1. 色相一致的配色，会让人同时感到统一感，容易让人产生联想，更具印象感。
  2. 采用多种色相的时候常常可以用纯度一致来统一配色。纯度低的配色让热你感觉稳重，纯度高的配色让人感觉有精神。
  3. 明度一致的设计，能使整体有均衡而平面的印象。明度一致的配色，不会出现强弱和对比，因此，不太适用于需要隐忍瞩目的广告用途。

纯度差异可以产生不同的印象。

  1. 人们的目光容易被显眼的色彩所吸引，所以，高纯度的颜色常作为“强调色”。
  2. 纯度高的色彩有刺激人类感情的效果，面积越大，效果越明显。
  3. 避免过度使用纯度高的颜色，要想有效的使用，发挥其鲜艳的效果，就得压低其它色彩的纯度。
  4. 纯度低的配色会让人产生怀旧或平和的情感。与纯度高的色彩组合在一起会产生“戏剧性”的效果。

色相差异的对比，将未涉及带来冲击力和华丽感。

  1. 不同色相所形成的对比效果，是以色相环中的位置距离而定，距离越远，产生的效果越强烈。色相环中，最强烈最具冲击力的便是相对位置的“互补色”。设计界上的撞色便由此而来。
  2. 若以互补色配色，又是高纯度的，变回引起“光晕”现象，使得文字难以观看，此时，必须降低一方色彩的纯度。（光晕现象：在色与色的边界产生其他颜色）
  3. 将不同色相的颜色相组合时，色相感觉也会不同。也就是，即使颜色相同，只要周围的颜色改变，看起来感觉也会不同。

无彩色，指完全没有颜色的色彩，也就是常说的黑白灰。

  1. 无彩色最大的特征，就是与其它色彩组合时，不会使对方色彩印象产生变化，不会与其冲突。
  2. 无彩色具有整合设计视觉的功能。例如许多明度、纯度不一的色彩，只要背景采用无彩色，就能形成整体感。
  3. 黑色具有收敛设计印象的效果，与黑色搭配，色彩会看起来纯度更高。

诱目性。纯度高的暖色系色彩容易映入眼帘，被称为“诱目性”高的配色。

根据诱目性可以做到一个简单的引人注目的配色方案：

  1. 背景使用纯度低的冷色系色彩。
  2. 要突显某处，请用纯度高的红色或黄色。
  3. 主体加上外框线来强调出明度差。

不过，色彩的诱目性只是暂时的。若长时间观看这种配色，眼睛和大脑就会感觉到疲倦。所以这种配色并不适用于阅读文本等需要仔细阅读的地方。

色彩也具有节奏！如同音乐一般，要思考选择什么色彩，如何组合，面积和位置摆放都会产生“节奏”，形成不一样的效果！

运用

看完这本书后，想起了刚开始实习的时候，做的一个网页设计。因为是做绿色能源的，采用深绿色为主体色调，初稿做完后在配色上总觉得很别扭，却道不出原因。而后将背景色改为常规的白色后，才显得比较自然。

自己总结出了原因，深色是用来突出显示一些重要但不需要长时间注目的信息，而显然这个网页区块需要的不是这个，因为产品库、价格走势图这些都是会长时间看的，深色容易造成视觉疲劳。

终稿看上去就好多了。

运用书上的知识，是因为绿色显然诱目性比较高，而且面积大提高了纯度的印象（影响力），长时间阅读会显得疲倦。

小结

显然一本书代表不了什么，色彩这块还有很多地方需要学习和实践，保持一颗学习的心:)

思想碰撞——反应堆网站设计

反应堆是一个用于发起临时聊天、线上会议的应用型网站。

采用扁平化设计，拥有弱用户化、会议流程控制、会议记录等特色功能。

更多设计图

灵感来源

刚开始只是前端的小伙伴伟楠卖弄新学的node.js做“实时推送”，问我有没有设计让他练练手。于是我用空闲时间想，到底实时推送能做什么，发现该有的，互联网上已经有人做了，想了很久终于有了这个做临时聊天、线上会议的想法。

从想法萌生到头脑风暴、整理交互功能就花了近一个星期，然后开始设计，每一天晚上看着自己完成的设计稿，总觉得不满意，第二天再改。

开始设计得很繁杂，后来不断删减改进，最后做到第六稿才满意地完成。

交付完设计稿后，成就感妥妥的~

特色之处

这算是一个产品，那么它到底有什么功能特色能够击败同类型产品呢。

在头脑风暴时，列出的几个同类型产品：qq，微信，飞信。他们各有优点，也很强大。但针对这个线上会议领域，缺点也很多的。

首先，qq、微信等需要添加好友才能发起聊天，现在很多人拥有一种“社交压力”不愿意添加好友。而反应堆通过链接、代号形式邀请人员，这种弱用户化提供方便的同时也免除了这种烦恼。

其次，qq、微信建立群、讨论组步骤繁琐，且聊天氛围不够专业，过多的“恶搞”聊天表情影响气氛。同时有来自于其他联系人，广告等干扰。而反应堆采用扁平化的简洁界面，针对性的功能，提供一种大气的线上会议气氛。

最后，反应堆拥有实时推送消息、跨平台、会议流程控制、用户状态显示、笔记记录共享等特色功能，专门为线上会议而生。

感想

很多时候，我们需要灵感迸发。但常常事与愿违，因为这不是临时就能拥有的。

就我而言，平时关注互联网技术、信息，这算是一个积累过程。而后在工作室这个轻松自由的环境氛围下，更能激发出我们的创造力。

同时，灵感很多，实践很少也是一个问题，做完这个网站设计后，我还想完成之前构想的一个音乐播放器和工作室oa客户端设计。同时这是针对移动App的设计，弥补一下这块的不足。

现在初期还不完美，后面继续完善吧。

回想自己与设计

昨天晚上在工作室我们一群大三的敲代码的、做设计的跟师兄师姐们聊了一晚上，我事先没准备什么问题，只是想到什么就问什么。对于接下来找实习面试，每个人都在全力准备着啊。在此之际，回想了下我的设计之路，挺值得回味的。

开发与设计

关于开发与设计，这似乎是两项不相容的事情，一个偏向理性，一个偏向感性，但却有不少两者兼爱好的人，如我。设计，翻出我两年前面试工作室的简历，个人评价上大概会看到“我热爱设计”这个字样。是啊，那时，不，从小学开始，我就不差这份热情了吧。我面试的是flash组，记得师姐面试时问我，你热爱设计为什么不去面试平面组吖？我说因为在flash组也同样可以发挥我的设计，而我也喜欢编程做自己的游戏，自己设计自己开发。事实上，两年多来虽然一直学编程，但从未停止对设计的热爱，一直关注着互联网设计这个圈子。

现在想起自己高中“稚嫩”的设计，那时的设计真的很挫，但自己已经开始“设计”，开始让灵感迸发形成“自以为”美的东西，并且沉醉于此，那时设计logo一做就是大半天，没有经验，没有老师，就这样自己在电脑前做着玩但很满足～

真感慨那时候自己不知从哪来的自信，让我认为自己设计很有天赋，大概是初生牛犊不怕虎吧。

而现在眼界开阔了，也意识到自己的渺小与水平有所缺陷。不过还是喜欢自己的设计风格和套路，首先是对自己的设计要有自信，同时在清楚自己的水平的情况下，不得过傲。最重要的一点就是，懂得学习，懂得使用工具，做到“高效”设计。

另外，做设计还需要有自己的思想，有自己的灵魂，发挥自己的创造力。我觉得设计是一个走心的过程，一般看一个人的设计作品，就能反映一个人的思想性格。不像编程，纯技术，纯脑力逻辑思维。

绘画

我的绘画并不出众，只能说还行。

以下是一些小学的趣事了，可以略过。

小学开始，涂鸦，有几件事那时很让我得意一阵子。

一年级还是幼儿园的时候我画房子是画出了3D外加透视的！而那时纵观身边的小伙伴们都在画二维平面的。现在想来不知道是不是那时他们还没理解到三维，跟他们卖弄的时候都完全无视……

三年级的时候我很喜欢设计“艺术字体”——那时我很热衷将英语书上的字体3D化，加阴影加投影搞特效（火焰、木头啊）。说到英语书，上面的Tom和Mary已经被我改编成很多连环画，以及翻页动画，西游记各种剧情。

那时发现自己的一个新技能，临摹美术书上的作品每次都能达到八九成的相似度，自此一直对自己的绘画能力很有自信。对绘画的爱好一发不可收拾，也得到老师的认同，参加学校的绘画特训组，画些水墨画和素描，不过那时还是玩玩。

也有让我吃瘪的时候，小学有次参加比赛给“蓝猫淘气”卡通人物线稿涂色，我充分结合了油画、写实的阴影、投影的混色涂法（也就是看美术书上的作品看多了)。结果大家都说远看很立体很像，近看很“脏”，普通的平面涂色好看多了。让我郁闷了好一阵子。

初中开始尝试写实绘画，包括素描，但时间不多，大多是为了美术课的作业，初中时从初一开始就得到美术老师不少赞扬，最深刻的是初一时第一次美术作业我的“印象派伪写实”画的风景画，老师的点评让我很是得意。还有一次我设计的一个黑白无常面具（有暗格拉绳能够手动吐舌头的！！），让老师拿去美名其曰拿去收藏给他家小孩玩去了T^T。

到了高中就没有美术课，但也会独自涂鸦，高一也有临摹动漫图（例如很小张的海贼王的动漫贴纸，照着我能放大好几倍画到A4纸上，相似度到九成，以致于有很多女生找我画～）。后来开始慢慢转变风格了，从写实派到印象派，从抽象到超现实，也不知什么原因（青春躁动期！），就是对写实有点厌倦了，就是觉得抽象派和印象派绘画更能寄托、表达我的情感。

其实呢，至今想起有件事让我很憋屈无奈的，就是小学时我第一次拿我得意的绘画作品给爸看时，然后——完全无视！大概是他觉得除了学习成绩外，这些都是娱乐吧，尔后我就没拿过给爸妈看了。初中时，逢学期末老师评价都会在学生评价那附上句我绘画天赋不错，结果爸妈也无视……其实是我初中成绩很不错，就不想我学其它了。

设计

真正接触“设计”是在高中吧，自学了ps、ai、flash软件。粗浅的设计logo、VI（那时猪八戒网站刚刚兴起）。而到了大学，加入社团、工作室，师兄师姐带着入门，开始对设计有新的领悟，持续关注互联网设计，加上图书馆很多相关的书，这是设计也有了新一层的感觉。

所谓艺术，文学也是艺术，也有美的地方，建筑也是艺术，编程也是艺术，大自然也是。懂得了真正的主旨一法通，万法通，也就如此。

我觉得跳出设计这个领域能做出更好的设计。在关注设计的同时，也要有自己其它的兴趣爱好，例如我看看文学小说，画一下画，做做DIY电子。多在外界走走看看，多与人交谈对话，而不是仅仅盯着几本有关设计的书和ps窗口去做设计。

美好的东西能带给人们快乐。在生活中我们要多关注美的东西，不仅在视觉上，美是一种感觉，他品质很美，这件事很美，这段文字很美，等等，从其中获取灵感，领悟与设计。

都说设计如设计师的性格，我的性格是什么呢？我想，应该是随和吧：设计自由而随心。但不代表随便！这让我不停追寻新的设计风格，作出创新。有段时间在看文学小说时，一直在研究一种诗意的设计；还有在想象一种具有宏大感的微观设计。扁平化设计如今很火，其实感觉在11年扁平化流行之前我已在应用，因为那时候一直很喜欢矢量的鲜艳色彩纯色块设计，也觉得是一种偷懒高效的简洁美，。现在尝试研究如何在扁平化与拟物化之间作融合和突破。

后记

说回昨晚，最后回来的时候洁浩告诉我不要太紧张，水平是够的，说他们那边很多设计水平都不如我，我不知道那是安慰我还是鼓励我。哎，纵使我一向掩饰我的紧张与压力感，但还是让洁浩看穿了（捂脸）……

看着身边做设计的颖欣和海玲，思绪回想起两年前第一次寒假实践时，我在一旁学习AS3编程，做完了不忘吐槽她们做一天还没做完几个button，我玩笑着展示下我用10分钟做完3个button还用flash做了交互的情景，不禁觉得当时很好笑……

flash开发的没落，终于还是让我回归了设计这条道路。

我的设计博客

为了我的设计博客，前几天建的Lofter博客，今天终于布置好了。

不得不说，网易的Lofter轻博客做得很好，各种自定义功能与模块,管理也很方便快捷。其中博客模板就有很多，而且很漂亮，不过不能随意修改，只能通过其设定好的自定义项修改。

今天挑来挑去，不是差这个不够理想，就是差那个不够美观。最后，终于确定了一个，响应式设计的，没有什么重大缺点也没有什么亮点的模板。

博客名字叫伴星流形，自己还是有点文学情节，起了这个名字，伴星是我一直以来用的网名，流形是源于文天祥的《正气歌》：

天地有正气，杂然赋流形。下则为河岳，上则为日星。

另外还有易经《彖》：

大哉乾元，万物资始，乃统天。云行雨施，品物流形。

流形，字面上解释起来就是：各种形态，指天地宇宙间的一切形体。

一经看到这个词，我就喜欢上了它了，流形流形，设计不就是这样吗？上善若水，大道无边。任何领域从技术升到艺术层面就应该不再有约束存在了，这和我高一的语文老师说的差不多，原话忘了，大致意思是：有些同学，写的作业以为自己在写狂草，字难看得我看都不想看。你们现在写字就要一个一个写工整，等你达到更高境界了，才能随心所欲，任笔挥写，讲求意到笔随。

所以博客名叫伴星流形，英文名取流形翻译叫Manifold。愿我的设计能有星空的虚空磅礴，亦有流形般不受约束，尽情绽放自己的创想。

但愿如此。
