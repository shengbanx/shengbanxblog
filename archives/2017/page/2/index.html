设计要闻-2017.2

17年2月设计要闻，月中一篇Behance 2017设计趋势传得火热，很长一篇，不过讲得全的同时也很泛，私以为趋势这种东西，仁者见仁，智者见智。不过在国内外众多趋势类总结文章中，有些的观点还是很独到的，例如国内大宝的一篇趋势预测。

值得一提有一篇介绍UI组件管理器思路的文章，重新唤起我对UI设计效率提升的构想。将现有的设计规范更进一步，成为所见即所得的“组件库”，提高设计效率。

另外，网站在移动端的响应式处理已经存在很久，但是少有人提到通常网站的这种响应式处理会让其在移动端中显得过长，如何较好地处理这个问题，值得一番研究。

最后，如果了解的话应该会知道，程序员热衷一种叫做 git 的项目版本控制工具，但设计师却很少会用上，因为psd等设计源文件太大，很难进行上传和回档操作。Abstract这个新软件的出现或许会有些期待吧，虽然目前只能是完美支持Sketch。真希望某一天adobe自家出一个PS/AI的版本控制功能。

设计要闻-2017.1

17年1月设计要闻，这个月涌现了很多关于AI（人工智能）如何影响着设计的热点。对比淡下去的VR设计，这可能更值得关注。关于人工智能，当然不可能取代设计师，而如果仅仅作为辅助设计的话，我觉得不应该对其排斥，至少Adobe MAX大会上谈及的很多项目都与此有关。可能你现在就使用着Adobe系列软件中相关的功能而并未发觉呢。

PaintChainer就是一个很有趣的工具，只需上传线稿并标记上色区域，就可以自动上色。这是一位日本的工程师在15年6月公开的深度学习框架Chainer，还有Python借口的OpenCV（Intel的视觉库），通过深度学习网上60万张已上色的插画师作品来学习上色方式。有兴趣可以看下知乎相关的讨论。

目前设计工具的数量也变得多了起来，虽然我们使用最多的还是PS、AI这些，但不妨多了解一下，《2017年设计工具形势》这篇文章中就有分析讲述。

另外，Dribbble 被 Tiny 收购，这几年Dribbble一直不温不火，推出的新功能也只有Playbook，貌似去年说好的Training栏目也一直未推出。对Dribbble团队来说，被收购以后或许可以双赢发展得更好吧。
